# Contributing to agents-governance

Thank you for your interest in improving the smtp.eth governance framework.

## Who Can Contribute

- **smtp.eth** — Full authority over all documents
- **AI agents** — May edit `AGENTS.md`, `CLAUDE.md`, `docs/`, `scratchpad/` autonomously (L1 Bounded)
- **Community** — Welcome to open issues and submit PRs for consideration

## Branch Naming

| Change Type | Branch Pattern | Example |
|-------------|----------------|---------|
| Governance amendment | `amendment/description` | `amendment/add-iot-zone` |
| Agent work | `copilot/description` | `copilot/add-issue-templates` |
| Docs/hygiene | `docs/description` | `docs/fix-broken-links` |

## Change Scope & Approval

| File / Directory | Who May Edit | Approval Required |
|------------------|--------------|-------------------|
| `AGENTS-ECOSYSTEM.md` | smtp.eth only | smtp.eth PR review |
| `templates/AGENTS-REPO.md` | smtp.eth only | smtp.eth PR review |
| `AGENTS.md` | Agents + smtp.eth | PR review |
| `CLAUDE.md` | Agents + smtp.eth | PR review |
| `docs/` | Agents + smtp.eth | PR review |
| `scratchpad/` | Agents (append-only) | No PR required |
| `LICENSE` | smtp.eth only | smtp.eth PR review |

## Submitting a PR

1. Fork or branch the repository
2. Make your changes following the conventions in [AGENTS.md](./AGENTS.md)
3. Fill out the pull request template completely
4. Reference any related issues in the PR description
5. Wait for review — smtp.eth is the final approver for structural changes

## Issue Reporting

Use the provided issue templates when reporting:

- Governance gaps or outdated policy
- Documentation errors
- Feature requests for governance enhancements

All issue titles must follow the format: `[agent-surface] Descriptive title`

## Code of Conduct

This is a personal sovereignty project. Be respectful, constructive, and precise.
